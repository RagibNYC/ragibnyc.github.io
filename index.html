proace_revenue_etl.sql ProACE International · Data Pipeline 
-- ETL Pipeline: Multi-source revenue reconciliation
-- Unified source systems → achieved 100% financial data accuracy
-- ProACE International | Mar–Apr 2024

WITH crm_revenue AS (
    SELECT
        client_id,
        SUM(deal_value)                                    AS crm_total,
        COUNT(*)                                           AS deal_count,
        MAX(close_date)                                    AS last_close
    FROM crm_system.deals
    WHERE status = 'closed_won'
      AND close_date >= '2024-01-01'
    GROUP BY client_id
),
erp_invoices AS (
    SELECT
        client_id,
        SUM(invoice_amount)                                AS erp_total,
        SUM(CASE WHEN paid_date IS NOT NULL THEN invoice_amount ELSE 0 END) AS collected
    FROM erp_system.invoices
    GROUP BY client_id
),
variance_check AS (
    SELECT
        c.client_id,
        c.crm_total,
        e.erp_total,
        c.crm_total - e.erp_total                         AS variance,
        ROUND((c.crm_total - e.erp_total) / c.crm_total * 100, 2) AS variance_pct
    FROM crm_revenue c
    LEFT JOIN erp_invoices e ON c.client_id = e.client_id
)
SELECT
    v.*,
    CASE
        WHEN ABS(variance_pct) < 0.5  THEN 'RECONCILED ✓'
        WHEN ABS(variance_pct) < 2.0  THEN 'REVIEW NEEDED'
        ELSE                               'FLAG FOR AUDIT'
    END                                                    AS reconciliation_status,
    ROW_NUMBER() OVER (PARTITION BY CASE WHEN ABS(variance_pct) >= 2 THEN 1 ELSE 0 END
                         ORDER BY ABS(variance) DESC) AS priority_rank
FROM variance_check v
ORDER BY ABS(variance) DESC;
What this did: This query ran nightly. Any variance above 2% was auto-flagged to senior leadership before the morning report. Reduced manual reconciliation from 3 days to 3 hours. Financial data accuracy went from 67% → 100%.
revenue_time_intelligence.dax Power BI Dashboard · ProACE International 
-- DAX Time Intelligence Measures
-- Used in automated KPI dashboard to replace manual Excel reporting

-- YTD with prior year comparison for variance analysis
Revenue_YTD =
CALCULATE(
    SUM( FactRevenue[Amount] ),
    DATESYTD( DimDate[Date] )
)

Revenue_PriorYTD =
CALCULATE(
    [Revenue_YTD],
    SAMEPERIODLASTYEAR( DimDate[Date] )
)

-- Budget vs Actuals variance with % and direction flag
Variance_vs_Budget =
VAR ActualAmt  = SUM( FactRevenue[Amount] )
VAR BudgetAmt  = SUM( FactBudget[BudgetAmount] )
VAR VarAmt     = ActualAmt - BudgetAmt
RETURN
IF(
    ISBLANK( BudgetAmt ),
    BLANK(),
    DIVIDE( VarAmt, ABS( BudgetAmt ) )
)

-- KPI status for traffic-light scorecard (used in conditional formatting)
KPI_Status =
SWITCH(
    TRUE(),
    [Variance_vs_Budget] >= 0.05,   "BEAT ↑",
    [Variance_vs_Budget] >= -0.02,  "ON TRACK →",
    [Variance_vs_Budget] >= -0.10,  "AT RISK ↓",
                                      "MISS ↓↓"
)
What this did: These measures powered the automated KPI dashboard that replaced separate weekly Excel reports. The traffic-light KPI_Status measure drove conditional formatting that allowed leadership to review metrics rapidly.
dcf_model_formulas.xlsx — Key Formulas J.P. Morgan · M&A DCF Model 
// ─── DCF Model — Key Excel Formula Logic ───────────────────────────
// Built for $1B EBITDA M&A target · J.P. Morgan Virtual Experience
// Three-statement model: IS → BS → CFS → Free Cash Flow → DCF

// Revenue Forecast with scenario toggle (Bear=0.88x, Base=1.0x, Bull=1.15x)
Revenue_Y1 = Revenue_Base
           * CHOOSE(ScenarioIndex, BearGrowth, BaseGrowth, BullGrowth)

// EBITDA with margin expansion assumption
EBITDA_Y1  = Revenue_Y1
           * (BaseEBITDA_Margin + MarginExpansion * (Y - 1))

// Free Cash Flow to Firm (FCFF)
FCFF       = EBITDA
           * (1 - TaxRate)
           + D_and_A
           - CapEx
           - Change_in_NWC

// Present Value of each year's FCFF (discounted at WACC)
PV_FCFF_Yn = FCFF_Yn / (1 + WACC) ^ n

// Terminal Value — Gordon Growth Model
TV         = FCFF_Y5 * (1 + TGR) / (WACC - TGR)
PV_TV      = TV / (1 + WACC) ^ 5

// Enterprise Value
EV         = SUM(PV_FCFF_Y1:PV_FCFF_Y5) + PV_TV

// WACC Sensitivity Table (drives the heat map matrix)
EV_Sensitivity[r,c] = SUM_of_PV_FCFFs(WACC_Array[c])
                    + PV_TV(WACC_Array[c], TGR_Array[r])
What this did: The WACC sensitivity matrix (5×5 = 25 EV data points) was built with a single formula dragged across the table — no hardcoded values. Changing one input cell updated all 25 enterprise values simultaneously.
